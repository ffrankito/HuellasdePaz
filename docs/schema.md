# Database Schema

The CRM uses Supabase (PostgreSQL) with Drizzle ORM.

Full UML diagrams (use cases, ER, sequence, state): [`crm/docs/diagramas-uml.md`](../crm/docs/diagramas-uml.md)

---

## Core Tables

| Table | Description |
|---|---|
| `usuarios` | Internal team members (role, permissions) |
| `clientes` | Pet owners — includes `token_portal` for portal access |
| `mascotas` | Pets — includes gallery, dedication, public memorial flag |
| `servicios` | Cremation/burial services — 9-state lifecycle |
| `planes` | Contracted prevention plans |
| `planes_config` | Plan templates |
| `leads` | Prospects from landing/cotizador/direct/veterinaria |
| `lead_interacciones` | Lead interaction history |
| `inventario` | Urns, bags, accessories, supplies |
| `convenios` | B2B agreements (vets, petshops, shelters) |
| `templates_msg` | WhatsApp/email message templates |
| `comunicaciones` | Sent communications log |
| `configuracion_general` | System settings (key/value JSONB) |
| `noticias_cementerio` | Cemetery news visible in the client portal |

## Service Lifecycle

```
pendiente → en_proceso → listo → entregado
                               ↘ cancelado
```

## Lead Lifecycle

```
nuevo → contactado → interesado → cotizado → convertido
                                           ↘ perdido
```

## Migration History

Migrations 0000–0009 are generated by drizzle-kit and tracked in `crm/supabase/migrations/meta/_journal.json`.
Migrations 0010+ are applied manually via SQL (not in the journal). See [`docs/setup.md`](./setup.md) for how to run them.
